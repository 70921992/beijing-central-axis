## 一、已修复：标记位置首次错误问题

### 根源分析

| 层级 | 旧方案 | 问题 |
|------|--------|------|
| 地图尺寸 | `mapImage.naturalWidth / naturalHeight` | 浏览器 WebP 解码引擎差异 → 不同终端返回值微差 |
| 标记定位 | `left: x%` 相对于 `naturalWidth` | `naturalWidth` 漂移 → 百分比点乘后像素级偏移 |
| 变换矩阵 | `transform: translate(ox,oy) scale(s)` | 基值漂移 → 缩放后偏移放大 |

**本质**：坐标系原点不稳——百分比 `x/y` 是精确的，但基向量 `naturalWidth/naturalHeight` 因浏览器而异。

### 修复方案

```javascript
// 修复前：动态读取（不稳定基线）
state.imgW = mapImage.naturalWidth;  // Chrome: 7087, Firefox: 7087, Safari×Retina: 可能不同
state.imgH = mapImage.naturalHeight;

// 修复后：硬编码基准（绝对一致）
var IMG_W = 7087;
var IMG_H = 7424;
state.imgW = IMG_W;
state.imgH = IMG_H;
```

**效果**：全终端坐标系统一，首次访问位置即正确，无需校准。

## 二、跨终端自适应增强策略（三层架构）

### 2.1 现状诊断

| 断点策略 | 当前状态 | 局限 |
|----------|----------|------|
| 基础断点 | `@media (max-width: 640px)` | 仅区分手机/桌面二档 |
| 无大屏增强 | `@media (min-width: 1200px)` 仅调字号 | 平板（768-1024）无覆盖 |
| 无设备特征检测 | 无 | 折叠屏、横屏无适配 |

### 2.2 三层自适应架构

```
┌─────────────────────────────────────────┐
│  L3: 设备特征感知层                      │
│  - prefers-reduced-motion（无障碍）       │
```

### 2.3 L2 弹性布局核心改造

```css
/* 旧方案：硬编码像素 */
.brand-text  { font-size: 21px; }
.detail-desc { font-size: 14px; }
.gallery-stage { height: 300px; }
.icon-btn { width: 36px; height: 36px; }

/* 新方案：clamp() 流体范围 */
.brand-text  { font-size: clamp(16px, 3vw, 28px); }
.detail-desc { font-size: clamp(11px, 1.5vw, 16px); }
.gallery-stage { height: clamp(180px, 35vh, 360px); }
.icon-btn { width: clamp(32px, 5vw, 44px); height: clamp(32px, 5vw, 44px); }

/* 移动安全区 */
#map-container {
    padding-left: env(safe-area-inset-left);
    padding-right: env(safe-area-inset-right);
    padding-bottom: env(safe-area-inset-bottom);
}

/* 动态视口（解决 iOS 地址栏隐藏/显示跳动） */
#app { height: 100dvh; }
```

### 2.4 L3 设备特征感知

```css
/* 无障碍：减少动画 */
@media (prefers-reduced-motion: reduce) {
    * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; }
}

/* 触屏设备：增大可点击区域 */
@media (pointer: coarse) {
    .icon-btn, .nav-btn, .side-nav-toggle { min-width: 44px; min-height: 44px; }
    .gallery-dot { width: 14px; height: 14px; margin: 4px; }
}

/* 横屏手机：详情卡片宽布局 */
@media (orientation: landscape) and (max-height: 500px) {
    .detail-card { flex-direction: row; max-height: 100vh; }
    .gallery-stage { width: 45%; height: 100%; flex-shrink: 0; }
    .detail-body { width: 55%; }
}
```

### 2.6 JS 增强：设备自动感知

```javascript
/**
 * 跨终端自动适配模块
 */
function detectDevice() {
    var w = window.innerWidth;
    var h = window.innerHeight;
    var dpr = window.devicePixelRatio || 1;
    var isTouch = 'ontouchstart' in window || navigator.maxTouchPoints > 0;
    var isLandscape = w > h;

    var tier = 'desktop';
    if (w < 360) tier = 'tiny';
    else if (w < 480) tier = 'phone';
    else if (w < 768) tier = 'phablet';
    else if (w < 1024) tier = 'tablet';

    document.documentElement.setAttribute('data-device', tier);
    document.documentElement.setAttribute('data-touch', isTouch ? '1' : '0');
    document.documentElement.setAttribute('data-dpr', dpr.toFixed(0));

    // 动态调整最小/最大缩放范围
    state.MAX_SCALE = isTouch ? 5 : 8;
    state.OVERVIEW_SCALE = w < 480 ? 0.9 : 1;
}

// 在 init() 和 resize 中调用
detectDevice();
window.addEventListener('resize', detectDevice);
```

## 三、分阶段实施建议

### 阶段 A：硬编码修复（已完成 ✅）

- IMG_W / IMG_H 常量化
- 版权 footer 字号 + 加粗

### 阶段 B：弹性布局改造（建议立即实施）

### 阶段 C：设备特征感知（中期改进）

### 阶段 D：极致适配（长期目标）

- 双屏 `window-segments` CSS
- View Transitions API 平滑切换

## 四、兼容性注记

| 技术 | 浏览器支持 | 降级策略 |
|------|-----------|----------|
| `clamp()` | Chrome 79+, Firefox 75+, Safari 13.1+ | CSS @supports 检测，回退固定值 |
| `dvh` | Chrome 108+, Safari 15.4+ | 回退 `vh` |
| `env(safe-area-inset)` | iOS Safari 11+, Chrome 69+ | 零值 fallback |
| `@media (pointer: coarse)` | 全主流 | 无兼容问题 |
| `viewport-fit=cover` | iOS Safari 11+ | 忽略 |
